Graph algorithms#

Graphs are incredibly versatile data structures and due to this versatility they are ubiquitous in computing. Note that for simplicity I while use \(n\) to denote the number of nodes in a graph instead of \(|V|\) and \(m\) to denote the number of edges in a graph instead of \(|E|\).

Graph traversal#

Depth first search (DFS)#

DFS explores the graph in a depth first manner meaning it explores all edges to unexplored nodes from a given node before backtracking to the node it started on once it reaches a node that has no edges that lead to unexplored nodes. In other words it recursively explores nodes until it reaches a leaf node at which point it backtracks. DFS uses a stack as it’s data structure. Essentially DFS operates like this

  1. Pick a node \(v\) to start and push it onto the stack

  2. Now pick one of its neighbors \(u\) to explore that you haven’t explored yet and push that onto the stack

  3. If \(u\) has no neighbors meaning it is a leaf node we pop it off the stack and go back to \(v\)

  4. If \(u\) does have neighbors pick one of its neighbors \(t\) to explore that you haven’t explored yet and push that onto the stack

Repeat this until you return back to \(v\).

Runtime is \(O(n+m)\)

DFS can be used to find connected components in undirected graphs since in an undirected graph DFS’s explore subroutine will explore all nodes reachable from the starting node and all nodes within the same connected component are reachable from any other node in the connected component in general for undirected graphs. Will label each node with it’s connected component number. When used on a DAG it returns a topological ordering of the nodes so can be used to topologically sort a graph.

Algorithm 9.11 (Explore)

procedure \(\text{Explore}(G, v,\) visited\()\):

Inputs Given a graph \(G=(V,E)\), a node \(v\) and a list of booleans, visited, which indicate which nodes have been visited

Output Nothing but visited(\(w\)) is set to \(\text{True}\) for all nodes, \(w\), which are reachable from \(v\)

  1. visited(\(w\)) = \(\text{True}\)

  2. \(\text{previsit}(v)\)

  3. for each edge \((u,v) \in E\):

    1. if visited(\(u\)) == \(\text{False}\):

      1. \(\text{Explore}\)(\(G\), \(u\))

  4. \(\text{postvisit}(v)\)

Algorithm 9.12 (Depth First Search)

procedure \(\text{DFS}(G)\):

Inputs Given a graph \(G=(V,E)\)

Output A list called visited which contains the pre-order and post-order number of each node computed during the depth first search.

  1. for each \(v\in V\):

    1. visited(\(v\)) = \(\text{False}\)

  2. for each \(v \in V\):

    1. if visited(\(v\)) == \(\text{False}\)

      1. \(\text{Explore}(G,v,\) visited\()\)

  3. return visited

Breadth first search (BFS)#

BFS explores the graph in layers. Returns the minimum number of edges one must traverse to get from a starting node \(s\) to all other nodes. If there is no path from \(s\) to a node \(t\) then the minimum number of edges one must traverse is set to \(\infty\). BFS utilizes a min heap (also called a priority queue).

Essentially BFS operates like this

  1. Pick a node \(v\) to start and push it onto the stack

  2. Now explore all of it’s neighbors

  3. Then pick one of its neighbors and explore all of it’s neighbors and do this for the rest of the neighbors as well

  4. Continue until the whole graph has been explored

This is why people say BFS explores the graph “in layers” the layers are the sets of neighbors in a sense.

Note that this means BFS will find the shortest paths in graphs that are unweighted but if the graph is weighted then the shortest path will in general not be found by BFS since each edge in general does not have the same weight.

Runtime is \(O(n+m)\)

Algorithm 9.13 (Breadth First Search)

procedure \(\text{BFS}(G, s)\):

Inputs Given a graph \(G=(V,E)\) and node \(s\in V\)

Output A list called dist which contains the distances from \(s\) to all nodes reachable from \(s\).

  1. for each \(v\in V\):

    1. dist(\(v\)) = \(\infty\)

  2. dist(\(s\))\(=0\)

  3. \(Q=[s]\) \(\quad \triangleright\) Create a queue holding just \(s\)

  4. while \(Q\) is not empty:

    1. \(u=\) dequeue(\(Q\))

    2. for each \((u,v) \in E\): \(\quad \triangleright\) Loop over all edges in graph

      1. if dist(\(v\)) == \(\infty\)

        1. enqueue(\(Q,v\))

        2. dist(\(v\)) = dist(\(u\)) + 1

  5. return dist

Shortest paths#

Dijkstra’s algorithm#

Sort of a generalization of BFS but works on weighted graphs where the edge weights are positive. If a negative edge weight exists Dijkstra’s will fail if this negative edge weight produces a negative weight cycle.

Runtime is \(O(n\log{(n)}+m\log{(n)})\)

Bellman-Ford algorithm#

Can be understood as a dynamic programming algorithm and works on graphs with edges weights that can be any real number so long as there are no negative weight cycles reachable from the starting node \(s\).

Runtime is \(O(nm)\)

Floyd–Warshall algorithm#

Can be understood as a dynamic programming algorithm. Runtime is \(O(n^3)\)

Strongly connected components#

Kosaraju’s algorithm#

This algorithm outputs a meta-graph where the nodes correspond to strongly connected components (SCCs) of the graph \(G\) and the edges \((u,v)\) denote that there is a path from all nodes in SCC \(u\) to all nodes in SCC \(v\).

Runtime is \(O(n+m)\) which is the same as DFS/BFS since we simply run DFS on \(G^R\) and DFS on \(G\).

Minimum spanning trees#

Definition 9.1

For an undirected weighted graph \(G=(V,E)\) a spanning tree is a (acyclic) subset of the edges \(T \subset E\) that connects all nodes and whose total weight is

\[ w(T) = \sum_{e\in T}w(e) \quad \text{where $e=(u,v)$} \]

i.e. the sum of the weights of each edge in the spanning tree \(T\).

It is said to “span” the graph \(G\) since it connects all vertices. The minimum spanning tree is a spanning tree for which \(w(T)\) is the minimal possible value for the graph \(G\).

Finding such an object for a given graph is useful for several applications such as electronic circuit design [Cormen et al., 2022]. Can we find the MST of an arbitrary weighted undirected graph efficiently? The answer is yes using a greedy algorithm. Note that the graph must also be connected or else there is no spanning tree by definition.

Kruskal’s algorithm#

The basic idea is to build the MST of the graph edge by edge while ensuring the edge you add at each step does not create any cycles (since we are building a tree).

Algorithm 9.14 (Kruskal’s Algorithm)

procedure \(\text{Kruskal}(G, w)\):

Inputs Given a connected undirected weighted graph \(G=(V,E)\) with edge weights \(w(e)\)

Output A minimum spanning tree given by the edges in \(T\)

  1. for \(v \in V\):

    1. make-set(\(v\)) \(\quad \triangleright\) Create a singleton set containing just node \(v\)

  2. Create \(E'\) which is a list of edges in \(G\) sorted in ascending order by edge weight

  3. \(T = \{\}\)

  4. for each edge \((u,v) \in E'\): \(\quad \triangleright\) Iterate over edges in sorted order

    1. if find-set(\(u\)) != find-set(\(u\)): \(\quad \triangleright\) Ensure \(u\) and \(v\) do not reside in the same set (to avoid cycles)

      1. \(T = T \cup \{(u,v)\}\)

      2. union(\(u\),\(v\)) \(\quad \triangleright\) Merge the sets containing \(u\) and \(v\)

  5. return \(T\)

Sorting the edges by weight takes \(O(m\log{(m)})\) time. Checking if an edge will create a cycle using the union-find data structure takes \(O(\log{n})\) time and we do this for all edges we iterate over so the loop in line 4 has time complexity \(O(m\log{(n)})\). Runtime is \(O(m\log{(n)})\) if \(n\geq m\).

Flow networks#

A flow network is a directed weighted graph \(G=(V,E)\) with start and end node, \(s\) and \(t\) respectively, where the edge weights \(c_e>0\) denote edge capacity i.e. the maximum amount of something that can flow through the edge \(e\in E\). For example in the flow network below if we were to imagine node \(s\) represented a water source and the edges denoted how much water in liters can flow along the edges, so \(3\) liters can flow form \(s\) to \(a\), \(2\) liters from \(a\) to \(d\) and so on, and node \(t\) is the node we wish to transport the water to.

A flow in the network is an assignment of flows, denoted \(f_e\), for each edge in the network such that \(f_e\leq c_e\) and the sum of flows into a node equals the sum of flows out of a node so the flow is conserved. For example in the flow depicted below we can see that a total of \(2+1+4\) units flow from node \(s\) and a total amount \(2+1+4\) units flow into node \(t\) the sink/target node.

Max-Flow problem#

The max-flow problem is the problem of finding an assignment of flows in the network such that the following constraints

\[\begin{split} \begin{align*} &f_e \leq c_e \quad \forall e \in E\\ \\ &f_v^{in} = f_v^{out} \quad \forall v \in V \quad \text{except nodes $s$ and $t$} \end{align*} \end{split}\]

are satisfied and the total flow out of the sink node \(s\), the source node, and therefore the total flow into \(t\), is maximized (so the max flow is equal to both the total flow out of \(s\) and the total flow into \(t\)). Note that the max flow is clearly upper bounded by which ever is greater between the sum of capacities of edges incident to \(s\) or sum of capacities of edges incident to \(t\).

Also note that cycles in the flow network do not affect the problem.

Ford-Fulkerson algorithm#

Find augmenting paths in the residual graph \(G^f\) for the current flow \(f\) using either BFS or DFS.

The runtime is \(O(Cm)\) where \(C\) is the size of the max flow. Note that this only holds if all edges have integer valued capacities so \(c_e \in \mathbb{Z^+}, \forall e \in E\). Also not that this runtime is similar to that of knapsack as it depends on \(C\) so it is not truly polynomial.

Edmonds-Karp algorithm#

Find augmenting paths in the residual graph \(G^f\) for the current flow \(f\) using BFS. Edge capacities can now be \(\in \mathbb{R}^+\).

The runtime is \(O(nm^2)\)

Other algorithms#

There have been faster algorithms found by James B. Orlin in 2013 which takes \(O(nm)\) time and by Li Chen et al. which takes \(O(m^{1+o(1)})\) time. Cool.

Max flow and min \(st\) cut#

An \(st\) cut is a partition of the vertex set of a graph into two sets \(L\) and \(R\) such that node \(s\) is in one set and node \(t\) is in the other. The capacity of such a cut is the sum of capacities of the edges that cross this cut i.e. the edges in the graph where one end resides on \(L\) and the other end resides in \(R\)

\[ capacity(L,R) = \sum_{(u,v)\in E| u \in L \text{ } \wedge \text{ } v \in R }c_{uv} \]

A min \(st\) cut is a cut that minimizes this capacity.

Theorem 9.4 (Max-flow min-cut theorem)

The minimum capacity of a cut in a flow network \(G\) is equal to the maximum flow in the flow network \(G\).

3.3 Topological ordering example#

a) To get this we first start with node A which will have preorder number 1 since it was the first to be visited. We then follow the edge to C which will have preorder number 2 since it was the second to be visited. Then we have two choices for which node to explore next, D or E, the problem says pick the one that is alphabetically first so we explore D which will get preorder number 3. We do this until we get to node G which will be the 5th to be explored so it gets preorder number 5. Now we backtrack by going back to node F and leaving node G so node G gets postorder number 6. From F we can now explore the other neighbor H which will get preorder number 7 and since it has no children we backtrack to node F and give node H postorder number 8. Since we explored all of F’s children we backtrack to D and give F postorder number 9 and so on until we get back to node A which gets post order number 14. Finally, we explore node B which gets preorder number 15 and since it has no children that haven’t been explored we stop the algorithm and give it postorder number 16.

The pre and post order numbers are:

Node A: pre-1, post-14

Node B: pre-15, post-16

Node C: pre-2, post-13

Node D: pre-3, post-10

Node E: pre-11, post-12

Node F: pre-4, post-9

Node G: pre-5, post-6

Node H: pre-7, post-8

b) sources are A and B. Sinks are G and H

c) Per the lecture we just need to sort the vertices by decreasing post order number to get the topological ordering found by the algorithm which is B,A,C,E,D,F,H,G.

d) There are \(2^3=8\) topological orderings for this graph. Since we have 2 choices for how to order A and B, two choices for how to order D and E and two choices for how to order G and H.

3.5 Reverse a graph#

Given a graph \(G=(V,E)\)

  1. Create a new empty adjacency list \(L\).

  2. Iterate over the nodes in the \(V\).

  3. When on node \(u\) check all of its neighbors. For a given neighbor \(w\) add \(u\) to the edge list of \(w\) in the new adjacency list \(L\).

At the end \(L\) will have all edges of \(G\) but reversed. This takes \(O(m+n)\) time since we must iterate over all nodes and their respective edges. Not sure if there is a way to do this in-place at the moment.

3.8 Pouring water#

3.11 Cycle containing an edge \(e\)#

If \(e\) is part of a cycle then removing it will result on the graph staying connected. If \(e\) is not part of a cycle (it’s a bridge) then removing it will result in the graph being disconnected. Knowing this we can simply run DFS and use it’s output to determine the number of connected components in \(G\) and then remove \(e\) from \(G\) and use DFS to find the number of connected components again. If the number increased by \(1\) then \(e\) was not part of a cycle. If the number stayed the same then we know \(e\) is part of a cycle.

Runtime is the same as DFS so \(O(n+m)\).

4.14 Shortest path through a given vertex#

Use Floyd-Warshall to find the shortest paths between all nodes. Then for each node \(v\in V\) find the shortest path found by Floyd-Warshall from it to \(v_0\) and the shortest path from \(v_0\) to every other node besides \(v\) and concatenate the paths to the shortest path from \(v\) to \(v_0\).

Complexity is \(O(n^3)\) since we used Floyd-Warshall and also have an additional \(O(m^2)\) component to create the shortest paths that pass through \(v_0\) for each node pair from the output of Floyd-Warshall.

For a faster way we can run Dijkstra on the graph with the source node as \(v_0\) which will give use the shortest path from \(v_0\) to all other nodes. Now for each node in \(G\) besides \(v_0\) we will run Dijkstra with it as the source node

Complexity is \(O(n(n\log{(n)}+m\log{(n)}))= O(n^2\log{(n)}+nm\log{(n)})\).

An even faster way is to run Dijkstra’s algorithm on the graph with the source node as \(v_0\) which will give use the shortest path from \(v_0\) to all other nodes. Then reverse the edges in the graph to make the graph \(G^R\). Now run Dijkstra’s algorithm on \(G^R\) with the source node as \(v_0\) this gives use shortest paths from \(v_0\) to all other nodes in the reversed graph but these paths (if we reverse the edges in them) will correspond to the shortest paths from all nodes to \(v_0\). Now for any two nodes \(w,z\) to get the shortest path from \(w\) to \(z\) that passes through \(v_0\) can be found by concatenating the shortest path from \(w\) to \(v_0\) found in our run of Dijkstra’s algorithm on the reversed graph with the shortest path from \(v_0\) to \(z\) found from our run of Dijkstra’s algorithm on the original graph.

Complexity is the same as that of Dijkstra’s algorithm since we run it two times and between runs create the reverse graph which takes \(O(n+m)\) time so the overall complexity is \(O((n+m)\log{n})\).

7.19#

First we check if the flow is valid so we quick if the flow along each edge does not exceed the capacity of that edge and we also check that for each node the flow into it = the flow out of it.

Next we construct the residual graph of the flow we were given. We then run BFS on the graph with \(s\) as the start vertex. If there is a path from \(s\) to the target node \(t\) in the residual graph then there is an augmenting path and thus the flow we were given is not optimal (maximum). If there is no path then we know the flow is a max flow.

Validating the flow takes \(O(n+m)\), creating the residual network takes \(O(n+m)\) and running BFS on it takes \(O(n+m)\) so our overall complexity is \(O(n+m)\).
